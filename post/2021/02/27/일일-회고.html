일일 회고

이동욱

2021/02/27

Categories: 회고 Tags: 회고

facts (사실, 객관)

  1. 오늘 개발 블로그를 만들었다.
  2. 오늘 회사에 일찍 일어나서, 출근을 하였다.
  3. 테스트 주도 개발 시작하기라는 책을 3장까지 읽었다.

feelings (느낌, 주관)

  1. 블로그 서비스를 이용하는 것보다는 많은 시간이 들었다.
  2. 아직 많은 기능이 필요하지만, 실제로 그 기능이 필요할 때 까지는 최대한 늦추도록 해야겠다.
  3. 일찍자고, 일찍 일어나는 것이 늦게까지 개발을 하다가 늦게 일어나는 것 보다 효율이 좋은 것 같다.
  4. 하지만, 회사에 출근을 했을 때, 무엇을 해야할지 계획을 세우지 않아서 조금 헤메였다.

findings (배운 점)

  1. 직접 개발 블로그를 만드니까, 조금 더 애정이 가는 것 같다.
  2. 일찍자고 일찍일어나니까 활용할 수 있는 시간이 훨씬 늘어난 기분이다.
  3. 테스트 코드에도 순서가 있다는 것을 알게 되었다.
  4. 테스트 코드의 중복을 제거할 때, 가독성을 해치지 않는지를 고려하고 조심스럽게 중복된 부분을 제거해야 한다.

affimation (자기 선언)

  • 나는 앞으로, 일찍 자고 일찍 일어나는 생활 패턴을 지킬 것이다.

회고 작성법

  1. facts(사실, 객관) 실제로 내가 했던 일이나 겪었던 일의 사실을 적는다.
  2. feelings(느낌, 주관) 내가 했던 일을 하면서 느꼈던 감정이나 느낌을 적는다.
  3. findings(배운 점) 내가 했던 일을 통해서 새롭게 배운 점이나 알게 된 점을 적는>다.
  4. affirmation (자기 선언) 내가 했던 일을 통해 배운 점과 아쉬운 점을 어떻게 유지 하고 개선할지를 적는다.
