마리아 DB Row Size too Large 에러

이동욱

2021/05/12

Categories: 개발 Tags: 데이터베이스

배경

  • 기존에 존재하던 테이블에, 컬럼 값을 추가하다가 마주친 에러이다.
Row size too large (> 8126). Changing some columns to TEXT or BLOB may help. In current row format, BLOB prefix of 0 bytes is stored inline.

  • 그러던 중에, 비슷한 현상을 겪는 사람이 커뮤니티에 글을 올려 놓았고, 원인을 찾을 수 있었다.

원인

  • 문제는 InnoDB에서 최대 로우 사이즈가 정해져있기 때문이다.

  • InnoDB에서 가변 길이의 컬럼은 오버 플로 페이지에 저장한다.
